# Git离线安装及内网使用



## 1 安装

### 1.1 Git 安装

`Windows` 直接使用安装包即可。`Linux` 下载源码进行编译。

默认安装路径 `/usr/bin` 可以直接在终端使用。

```shell
./configure
make && make install
```

### 1.2 配置本机Git属性

终端输入：

```shell
git config --global user.name "用户名"
git config --global user.email "邮箱"
```

这样设置全局的用户名称和邮箱，可以通过命令 `git config --global --list` 查看是否设置成功。

```shell
sudo vim ~/.gitconfig	# 配置文件路径
```



## 2 创建仓库（Linux）

### 2.1 环境

- Git
- SSH

`Git` 按照上述安装，`SSH` 服务目前系统一般自带。

### 2.2 前期准备

#### 2.2.1 开启SSH服务

查案状态

```shell
ps -e | grep ssh 		# 方法一
systemctl status ssh	# 方法二
```

开启服务

```shell
sudo /etc/init.d/ssh start
```

#### 2.2.2 创建用户

```shell
sudo groupadd git			# 创建 git 组
sudo useradd git -g git		# 创建 git 用户并添加进组
```

### 2.3 创建仓库

仓库分为裸露（bare）和非裸露，`bare` 模式只保存文件操作历史记录，仓库中是看不到具体文件。

#### 2.3.1 服务端创建普通仓库

##### 2.3.1.1 创建仓库

普通仓库在服务端可以查看到具体文件，但是不能进行修改因为 `Git` 中没有记录操作。

```shell
cd /home
sudo mkdir gitrepo	# 创建 git 仓库总目录
cd gitrepo
sudo mkdir test1	# 创建具体项目目录
cd test1
sudo git init		# git 非裸露初始化
sudo chown -R git:git /home/gitrepo	# 把仓库所属用户改为git 
```

##### 2.3.1.2 问题一：提交失败

这样创建的是包含工作目录，其他克隆仓库提交就会显示失败。

![](/media/image/2022-10-23-Git离线安装及内网使用/remote-reject.png)

需要修改服务端配置文件 `.git/config`。

```shell
# 方法一修改用户级，没作用
git config --global receive.denyCurrentBranch ignore 

# 方法二修改项目级
sudo vim .git/config
# 添加配置，允许使用同一分支
[receive]
	denyCurrentBranch = ignore
```

![](/media/image/2022-10-23-Git离线安装及内网使用/config.png)

##### 2.3.1.2 问题二：服务端不显示文件

需要修改服务端配置文件 `.git/hooks/post-update.sample`。将该文件后缀删除变为 `post-update`，`post-update` 注释 `exec git update-server-info` 并添加

```shell
unset GIT_DIR		# 取消 GIT_DIR 变量路径
cd ..				# 返回上级目录
git checkout -f		# 强制更新
```

![](/media/image/2022-10-23-Git离线安装及内网使用/post-update.png)

这样克隆仓库提交后，服务端工作目录也能查看文件。

#### 2.3.2 服务端创建bare仓库

`bare` 仓库在服务端无法查看到具体文件。只保存文件操作，文件以二进制形式保存。

```shell
cd /home
sudo mkdir gitrepo	# 创建 git 仓库总目录
cd gitrepo
sudo mkdir test2	# 创建具体项目目录
cd test1
sudo git init --bare				# git 裸露初始化
sudo chown -R git:git /home/gitrepo	# 把仓库所属用户改为git 
```



## 3 仓库使用

### 3.1 克隆

```shell
git clone git@192.168.1.5:/home/gitrepo/test2
```

### 3.2 上传

```shell
git add -A 				# 全部上传
git commit -m "commit"	# 备注
git push				# 提交
```

### 3.3 下载

```shell
git pull
```

### 3.4 远程关联

本地仓库初始化并远程关联

```shell
git init
git remote add origin git@192.168.1.5:/home/gitrepo/test2
```
